Этот маленький гайд про то, как поднять k3s кластер на базе физических или виртуальных хостов.
K3s — это, как сама компания предполагает, сертифицированный дистрибутив K8s для IoT и edge устройств (Подробно на сайте [k3s.io](k3s.io))

Почему использовать k3s, а не реальный k8s с kubeadm? Ответ прост его легко и быстро развернуть + он хорошо работает на слабых устройствах.

Так что поехали.
Сначала надо подготовить среду, я использую Debian minimal но можно выбрать любой дргой дистро.

После установки ОС настройте сети, чтобы все ваши устройства пинговали друг друга (не буду вдаваться, как это сделать читайте предыдущие гайды, я там подробно описывал).
Также для удобства рекомендую настроить ssh-сервер на каждой машине, чтобы быстро переключаться между ними.

```
systemctl start sshd
```

Если работаете в одном терминале, для вас хосты не различаются. Чтобы не путаться, назначьте имена хостов. Для этого откройте `/etc/hostname`, поменяйте содержимое на worker или master в зависимости от хоста (у меня 1 мастер-нода и 2 воркер-ноды).
Следующим шагом у нас будет обновление системы без этого никуда: `apt update`, `apt upgrade`.
Теперь идём на мастер и скачиваем k3s.
По задумке авторов вот так он должен установиться без проблем, но у меня так не сработало:

```
curl -sfL https://get.k3s.io | sh -s - server \
  --node-ip=192.168.100.10 \
  --advertise-address=192.168.100.10 \
  --bind-address=192.168.100.10 \
  --node-name=master
```

Поэтому я сохранил отдельно сам скрипт и запустил (т.е. если пойдёте на сайт [get.k3s.io](get.k3s.io), то увидите bash-скрипт, можете просто скопировать в .sh файл и использовать без проблем).

Также поменяйте адрес на адрес вашего мастера.

```
chmod +x main.sh

./main.sh server \
  --node-ip=192.168.100.10 \
  --advertise-address=192.168.100.10 \
  --bind-address=192.168.100.10 \
  --node-name=master
```

После установки проверяем, всё ли работает правильно:
```
sudo k3s kubectl get nodes
```
Вы должны увидеть мастер-ноду.

Далее перейдём к воркерам. Но чтобы их подключить, нам потребуется токен, его можно посмотреть в `cat /var/lib/rancher/k3s/server/node-token`, сохраните его в отдельном месте.
Перейдите на воркер-хост и запустите:
```
export K3S_URL=https://192.168.100.10:6443 
export K3S_TOKEN="ваш токен"
```
Здесь адрес сервера и порт, на котором у нас работает k3s, и также ваш токен.

```
./main.sh agent --node-ip=192.168.100.11 --node-name=worker1
```
А здесь IP воркера.

Также проверните с другим хостом, не забудьте поменять IP и имя.

(Так можно провернуть с любым количеством хостов, хоть 100. Но для оптимальной работы кластера нужно как минимум 3 хоста.)

И всё, можете посмотреть на свои ноды и поды:

```
kubectl cluster-info
kubectl describe node worker1
kubectl get all -A
```
