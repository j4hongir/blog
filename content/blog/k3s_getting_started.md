Этот небольшой гайд о том, как быстро поднять k3s-кластер на базе физических или виртуальных хостов.

## Что такое K3s и зачем он нужен?

K3s — это сертифицированный дистрибутив Kubernetes для IoT и edge-устройств, как заявляет сама компания Rancher. Подробнее можно почитать на официальном сайте [k3s.io](https://k3s.io).

**Почему K3s, а не полноценный K8s с kubeadm?**

Ответ прост: K3s легко и быстро разворачивается, плюс отлично работает на слабых устройствах. Он потребляет меньше ресурсов, но при этом остаётся полноценным Kubernetes-кластером.

Так что поехали!

## Подготовка окружения

Я использую Debian minimal, но вы можете выбрать любой другой дистрибутив Linux.

После установки ОС настройте сетевое соединение между всеми устройствами. Подробную инструкцию по настройке вы найдете в моих предыдущих гайдах.

Для удобства настройте SSH-сервер на каждой машине, чтобы быстро переключаться между ними:

```bash
systemctl start sshd
systemctl enable sshd
```


Если работаете в одном терминале, хосты не будут различаться. Чтобы не путаться, назначьте понятные имена хостов.

Откройте файл `/etc/hostname` и поменяйте содержимое на `master` или `worker1`, `worker2` в зависимости от роли хоста. У меня в кластере 1 мастер-нода и 2 воркер-ноды.

или 
```bash
hostnamectl set-hostname master  # или worker1, worker2
```


Без этого никуда:

```bash
apt update
apt upgrade -y
```

## Установка K3s на master-ноду

Теперь переходим к установке K3s на мастер-ноде.

По задумке авторов, вот такой командой он должен установиться без проблем:

```bash
curl -sfL https://get.k3s.io | sh -s - server \
  --node-ip=192.168.100.10 \
  --advertise-address=192.168.100.10 \
  --bind-address=192.168.100.10 \
  --node-name=master
```

Но у меня так не сработало. Поэтому я сохранил скрипт отдельно и запустил вручную.


Если перейдёте на [get.k3s.io](https://get.k3s.io), увидите bash-скрипт. Можете просто скопировать его содержимое в `.sh` файл и использовать:

```bash
# Скачиваем скрипт
curl -sfL https://get.k3s.io -o install-k3s.sh

# Делаем исполняемым
chmod +x install-k3s.sh

# Запускаем с нужными параметрами
./install-k3s.sh server \
  --node-ip=192.168.100.10 \
  --advertise-address=192.168.100.10 \
  --bind-address=192.168.100.10 \
  --node-name=master
```

**Важно:** Поменяйте IP-адрес на адрес вашего мастера!


После установки проверяем, всё ли работает правильно:

```bash
sudo k3s kubectl get nodes
```

Вы должны увидеть мастер-ноду в статусе `Ready`:

```
NAME     STATUS   ROLES                  AGE   VERSION
master   Ready    control-plane,master   1m    v1.28.x+k3s1
```

## Подключение worker-нод

Теперь перейдём к подключению воркеров.


Чтобы подключить воркер-ноды, нужен токен. Его можно посмотреть на мастере:

```bash
cat /var/lib/rancher/k3s/server/node-token
```

Сохраните этот токен — он понадобится для каждого воркера.


Перейдите на воркер-хост и выполните следующие команды:

```bash
# Экспортируем переменные окружения
export K3S_URL=https://192.168.100.10:6443
export K3S_TOKEN="ваш_токен_с_мастера"

# Запускаем установку агента
./install-k3s.sh agent \
  --node-ip=192.168.100.11 \
  --node-name=worker1
```

Где:
- `K3S_URL` — адрес мастер-сервера и порт (по умолчанию 6443)
- `K3S_TOKEN` — токен, который мы получили на мастере
- `--node-ip` — IP-адрес воркера
- `--node-name` — имя воркер-ноды


Повторите те же шаги на втором воркере, не забыв поменять IP и имя:

```bash
export K3S_URL=https://192.168.100.10:6443
export K3S_TOKEN="ваш_токен_с_мастера"

./install-k3s.sh agent \
  --node-ip=192.168.100.12 \
  --node-name=worker2
```

**Примечание:** Так можно провернуть с любым количеством хостов, хоть 100. Но для оптимальной работы кластера рекомендуется минимум 3 ноды.

## Проверка кластера

Вернитесь на мастер-ноду и проверьте состояние кластера:

```bash
# Информация о кластере
kubectl cluster-info

# Список всех нод
kubectl get nodes

# Детальная информация о конкретной ноде
kubectl describe node worker1

# Все ресурсы во всех namespace
kubectl get all -A
```

Вы должны увидеть все три ноды в статусе `Ready`:

```
NAME      STATUS   ROLES                  AGE   VERSION
master    Ready    control-plane,master   10m   v1.28.x+k3s1
worker1   Ready    <none>                 5m    v1.28.x+k3s1
worker2   Ready    <none>                 3m    v1.28.x+k3s1
```

## Полезные советы

Если хотите использовать kubectl без sudo, добавьте конфигурацию в ваш профиль:

```bash
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config
chmod 600 ~/.kube/config
```

Если нужно удалить K3s:

```bash
# На мастере
/usr/local/bin/k3s-uninstall.sh

# На воркерах
/usr/local/bin/k3s-agent-uninstall.sh
```
