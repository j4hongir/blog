# tcp header

|**Поле (Field)**|**Размер (Бит)**|**Логика и Назначение**|**Что там может быть / Контекст**|
|---|---|---|---|
|**Порт отправителя**<br><br>  <br><br>(Source Port)|16|Идентифицирует приложение/процесс на устройстве, отправившем пакет. Нужен для обратного ответа.|**0–65535**. Клиенты обычно используют «эфемерные» порты (например, 55315), серверы — «известные» (например, 80 для HTTP).|
|**Порт получателя**<br><br>  <br><br>(Destination Port)|16|Указывает, какому приложению на целевой машине предназначен пакет.|**0–65535**. Определяет службу (например, 443 — HTTPS, 22 — SSH).|
|**Порядковый номер**<br><br>  <br><br>(Sequence Number)|32|**Логика сборки:** Позволяет получателю собрать разрозненные пакеты в правильном порядке.<br><br>  <br><br>Если установлен флаг SYN — это начальное случайное число (ISN). Если нет — это смещение байтов данных.|Числовое значение. Гарантирует, что если пакеты придут не по порядку (2, 1, 3), получатель восстановит их как (1, 2, 3).|
|**Номер подтверждения**<br><br>  <br><br>(Acknowledgment Number)|32|**Логика надежности:** Сообщает отправителю, что данные получены. Число означает: «Я получил всё до этого байта, жду байт с этим номером».|Работает только если установлен флаг **ACK**. Значение = `Sequence Number` последнего принятого байта + 1.|
|**Длина заголовка**<br><br>  <br><br>(Data Offset)|4|Указывает, где заканчивается заголовок и начинаются данные. Измеряется в 32-битных словах (по 4 байта).|**Мин: 5** (20 байт, нет опций). **Макс: 15** (60 байт). Если значение 5, значит заголовок занимает 20 байт.|
|**Резерв**<br><br>  <br><br>(Reserved)|3-6|Зарезервировано для будущего использования. Должно быть обнулено.|Всегда **000**.|
|**Флаги управления**<br><br>  <br><br>(Flags / Control Bits)|9|**Логика управления состоянием:** «Рычаги», определяющие тип пакета (начало связи, конец, сброс и т.д.).|Каждый бит — переключатель (0 или 1):<br><br>  <br><br>• **NS, CWR, ECE**: Защита от перегрузки сети (ECN).<br><br>  <br><br>• **URG**: В пакете есть важные данные (см. Указатель).<br><br>  <br><br>• **ACK**: Поле «Номер подтверждения» валидно (почти всегда 1, кроме первого SYN).<br><br>  <br><br>• **PSH**: Протолкнуть данные приложению сразу, не буферизуя.<br><br>  <br><br>• **RST**: Аварийный разрыв (сброс) соединения.<br><br>  <br><br>• **SYN**: Синхронизация (начало соединения).<br><br>  <br><br>• **FIN**: Финиш (корректное завершение).|
|**Размер окна**<br><br>  <br><br>(Window Size)|16|**Логика контроля потока:** Говорит отправителю: «У меня в буфере свободно столько-то байт, больше не шли, пока не обработаю».|Число байт (0–65535). Если окно = 0, отправка прекращается до освобождения места. Позволяет медленному ПК тормозить быстрый сервер.|
|**Контрольная сумма**<br><br>  <br><br>(Checksum)|16|**Логика целостности:** Проверка на наличие ошибок при передаче (битые байты).|Вычисляется математически по заголовку и данным. Если сумма не сходится, пакет отбрасывается без уведомления.|
|**Указатель на срочные данные**<br><br>  <br><br>(Urgent Pointer)|16|Указывает смещение относительно `Sequence Number`, где заканчиваются «срочные» данные.|Работает только при флаге **URG**. Используется редко (например, в Telnet для прерывания команд).|
|**Опции**<br><br>  <br><br>(Options)|0–320 (бит)|Дополнительные параметры, не влезающие в основной заголовок.|**MSS**: Максимальный размер сегмента (обычно в первом SYN-пакете).<br><br>  <br><br>**Window Scale**: Увеличение размера окна (множитель).<br><br>  <br><br>**SACK**: Выборочное подтверждение (если потерян 1 пакет из 10, перешли только его).|
|**Данные**<br><br>  <br><br>(Payload)|Переменно|Полезная нагрузка. То, ради чего создавался пакет.|HTTP-запрос, часть картинки, фрагмент видео, текст email. Может отсутствовать (например, в пакетах рукопожатия `SYN` или `ACK`).|



# udp header

| --------------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Порт отправителя<br><br>(Source Port)**     | 16           | Идентифицирует приложение/процесс на устройстве отправителя. Нужен для обратного ответа (если требуется). | 0–65535. Может быть 0, если обратный ответ не нужен (например, в DNS-запросах иногда). Клиенты используют случайные порты (например, 54321).           |
| **Порт получателя<br><br>(Destination Port)** | 16           | Указывает, какому приложению на целевой машине предназначена датаграмма.                                  | 0–65535. Определяет службу (например, 53 — DNS, 123 — NTP, 67/68 — DHCP, 161 — SNMP, 5060 — SIP).                                                      |
| **Длина датаграммы<br><br>(Length)**          | 16           | Указывает полный размер UDP-датаграммы (заголовок + данные) в байтах.                                     | Минимум: 8 байт (только заголовок, без данных).<br>Максимум: 65535 байт (теоретически), практически ограничено MTU (~1472 байта данных при MTU 1500).  |

| **Данные<br><br>(Payload)**                   | Переменно    | Полезная нагрузка. То, ради чего создавалась датаграмма.                                                  | DNS-запрос/ответ, видеопоток, голосовой пакет (VoIP), игровые данные, DHCP-сообщения. Может быть пустым (только заголовок 8 байт).                     |
# ipv4 header

| **Поле (Field)**                                                    | **Размер (Бит)** | **Логика и Назначение**                                                                                   | **Что там может быть / Контекст**                                                                                                                                       |
| ------------------------------------------------------------------- | ---------------- | --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Версия**<br><br>  <br><br>(Version)                               | 4                | Идентификатор протокола. Маршрутизатор должен знать, как читать пакет.                                    | Всегда **4** (0100). Если там 6, пакет обрабатывается совсем по другой схеме.                                                                                           |
| **Длина заголовка**<br><br>  <br><br>(IHL - Internet Header Length) | 4                | Указывает, где заканчиваются служебные данные и начинается «нагрузка». Измеряется в словах по 4 байта.    | **Мин: 5** (20 байт). **Макс: 15** (60 байт). Обычно равен 5 (нет опций).                                                                                               |
| **Тип обслуживания**<br><br>  <br><br>(ToS / DSCP)                  | 8                | **Логика приоритета:** Указывает маршрутизаторам, как важно обработать этот пакет (очередь, задержка).    | **DSCP** (QoS): Например, голосовой трафик (VoIP) помечается как «срочный», а торренты — как «низкий приоритет».                                                        |
| **Общая длина**<br><br>  <br><br>(Total Length)                     | 16               | Полный размер пакета (Заголовок + Данные). Нужен, чтобы знать, где заканчивается этот пакет в потоке бит. | Максимум **65 535** байт. Если пакет больше, чем позволяет среда (MTU), он фрагментируется.                                                                             |
| **Идентификатор**<br><br>  <br><br>(Identification)                 | 16               | **Логика фрагментации:** Уникальный номер группы фрагментов. Помогает собрать разрезанный пакет обратно.  | Число. Если большой пакет разбили на 3 части, у всех трех будет **одинаковый ID**.                                                                                      |
| **Флаги**<br><br>  <br><br>(Flags)                                  | 3                | Управление «нарезкой» пакета. Разрешено ли резать и последний ли это кусок.                               | **DF** (Don't Fragment): «Не резать!». Если пакет не пролезает — выкинуть и сообщить ошибку.<br><br>  <br><br>**MF** (More Fragments): «Жди продолжения, это не конец». |
| **Смещение фрагмента**<br><br>  <br><br>(Fragment Offset)           | 13               | Показывает место текущего куска относительно начала оригинального пакета.                                 | Число. Например, «этот кусок начинается с 1480-го байта исходных данных». Нужен для склейки в правильном порядке.                                                       |
| **Время жизни**<br><br>  <br><br>(TTL - Time To Live)               | 8                | **Защита от зацикливания:** Счетчик «прыжков» (хопов) между роутерами. Уменьшается на 1 на каждом узле.   | Обычно 64 или 128. Если доходит до **0**, пакет уничтожается, отправителю летит ошибка (ICMP Time Exceeded).                                                            |
| **Протокол**<br><br>  <br><br>(Protocol)                            | 8                | **Диспетчеризация:** Сообщает, кому отдать данные внутри (какому протоколу верхнего уровня).              | **6** = TCP<br><br>  <br><br>**17** = UDP<br><br>  <br><br>**1** = ICMP (пинг)<br><br>  <br><br>**89** = OSPF                                                           |
| **Контрольная сумма**<br><br>  <br><br>(Header Checksum)            | 16               | Проверка целостности _только_ самого заголовка IPv4 (не данных).                                          | Пересчитывается на каждом роутере (так как меняется TTL). Если не сошлась — пакет молча удаляется.                                                                      |
| **IP отправителя**<br><br>  <br><br>(Source Address)                | 32               | Обратный адрес. Куда слать ответ или сообщение об ошибке.                                                 | Например: `192.168.1.50`.                                                                                                                                               |
| **IP получателя**<br><br>  <br><br>(Destination Address)            | 32               | Целевой адрес. Используется роутерами для выбора маршрута.                                                | Например: `8.8.8.8`.                                                                                                                                                    |
| **Опции**<br><br>  <br><br>(Options)                                | 0–320 (бит)      | Редко используемые служебные функции.                                                                     | «Запись маршрута» (каждый роутер дописывает свой IP), «Временная метка». Часто блокируются из соображений безопасности.                                                 |


# icmp header

| **Поле (Field)**                                           | **Размер (Бит)** | **Логика и Назначение**                                                       | **Что там может быть / Контекст**                                                                                                                                                                                         |
| ---------------------------------------------------------- | ---------------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Тип**<br><br>  <br><br>(Type)                            | 8                | **Главная категория сообщения.** Говорит «Что случилось?».                    | **0** = Echo Reply (Ответ пинга)<br><br>  <br><br>**8** = Echo Request (Запрос пинга)<br><br>  <br><br>**3** = Destination Unreachable (Не дошло)<br><br>  <br><br>**11** = Time Exceeded (Умер TTL)                      |
| **Код**<br><br>  <br><br>(Code)                            | 8                | **Уточнение причины.** Детализирует «Тип».                                    | _Для Типа 3 (Unreachable):_<br><br>  <br><br>**0** = Сеть недоступна<br><br>  <br><br>**1** = Хост недоступен<br><br>  <br><br>**3** = Порт закрыт (Firewall/нет службы)                                                  |
| **Контрольная сумма**<br><br>  <br><br>(Checksum)          | 16               | Проверка целостности самого ICMP-сообщения.                                   | Если повреждено — отбрасывается без уведомления.                                                                                                                                                                          |
| **Содержимое заголовка**<br><br>  <br><br>(Rest of Header) | 32               | Зависит от Типа/Кода. В диагностике используется для связки запроса и ответа. | **Identifier (ID):** ID процесса (чтобы знать, чей это пинг).<br><br>  <br><br>**Sequence Number:** Порядковый номер (1, 2, 3...) для отслеживания потерь.                                                                |
| **Данные**<br><br>  <br><br>(Data)                         | Переменно        | Полезная нагрузка или "улика".                                                | _При ошибках:_ Содержит заголовок IP + первые 8 байт данных оригинального пакета (чтобы отправитель понял, какой именно пакет вызвал ошибку).<br><br>  <br><br>_При пинге:_ Просто набор байт (напр., алфавит) для теста. |


# ipv6 header

| **Поле (Field)**                                            | **Размер (Бит)** | **Логика и Назначение**                                                                                           | **Что там может быть / Контекст**                                                                         |
| ----------------------------------------------------------- | ---------------- | ----------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| **Версия**<br><br>  <br><br>(Version)                       | 4                | Идентификатор протокола.                                                                                          | Всегда **6** (0110).                                                                                      |
| **Класс трафика**<br><br>  <br><br>(Traffic Class)          | 8                | Аналог ToS/DSCP в IPv4. Приоритезация.                                                                            | Разделение на чувствительный к задержкам трафик (VoIP) и обычный (Web).                                   |
| **Метка потока**<br><br>  <br><br>(Flow Label)              | 20               | **Новое:** Позволяет маркировать группу пакетов (поток) для одинаковой обработки роутерами без глубокого анализа. | Ускоряет обработку Real-time видео/аудио. Роутер видит метку и сразу шлет по кэшированному пути.          |
| **Длина данных**<br><br>  <br><br>(Payload Length)          | 16               | Размер "посылки" за вычетом 40 байт основного заголовка.                                                          | Включает в себя и данные верхнего уровня (TCP/UDP), и дополнительные заголовки (Extension Headers).       |
| **Следующий заголовок**<br><br>  <br><br>(Next Header)      | 8                | **Ключевая логика цепочки:** Указывает, что идет следом.                                                          | Либо протокол (6=TCP, 17=UDP), либо **Расширенный заголовок** (например, маршрутизация или фрагментация). |
| **Лимит переходов**<br><br>  <br><br>(Hop Limit)            | 8                | Аналог TTL в IPv4. Защита от зацикливания.                                                                        | Уменьшается на 1 каждым роутером. При 0 — сброс пакета + ICMPv6.                                          |
| **Адрес источника**<br><br>  <br><br>(Source Address)       | 128              | Откуда пришло. В 4 раза длиннее IPv4.                                                                             | Формат: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`.                                                        |
| **Адрес назначения**<br><br>  <br><br>(Destination Address) | 128              | Куда идет.                                                                                                        | Глобальный Unicast или Multicast адрес.                                                                   |


# ethernet header

|Поле (Field)|Размер (Бит)|Логика и Назначение|Что там может быть / Контекст|
|---|---|---|---|
|**Преамбула<br><br>(Preamble)**|56 (7 байт)|Логика синхронизации: Последовательность битов для синхронизации приёмника и передатчика. Позволяет "настроиться" на сигнал.|Чередующиеся 1 и 0: **10101010 10101010...** (повторяется 7 раз).<br>Не является частью кадра, но передаётся перед ним.<br>Помогает сетевым картам понять, что начинается передача.|
|**Начальный разделитель<br><br>(SFD - Start Frame Delimiter)**|8 (1 байт)|Указывает начало самого кадра. Последний байт синхронизации.|Всегда: **10101011** (последние 2 бита = 11 сигнализируют "дальше идёт кадр").<br>После этого байта начинается реальный Ethernet-кадр.|
|**MAC-адрес получателя<br><br>(Destination MAC Address)**|48 (6 байт)|Логика адресации: Физический адрес сетевой карты получателя. Определяет, кому предназначен кадр в локальной сети.|Формат: **XX:XX:XX:XX:XX:XX** (шестнадцатеричный).<br>Примеры:<br>• **00:1A:2B:3C:4D:5E** - уникальный адрес устройства<br>• **FF:FF:FF:FF:FF:FF** - широковещательный (broadcast, всем в сети)<br>• **01:00:5E:XX:XX:XX** - многоадресный (multicast, группе устройств)<br>Первые 3 байта (OUI) - производитель, последние 3 - уникальный номер.|
|**MAC-адрес отправителя<br><br>(Source MAC Address)**|48 (6 байт)|Физический адрес сетевой карты отправителя. Нужен для обратного ответа и идентификации источника.|Формат: **XX:XX:XX:XX:XX:XX**<br>Примеры:<br>• **A4:83:E7:12:34:56** - адрес вашего компьютера<br>• **DC:A6:32:AB:CD:EF** - адрес роутера<br>Должен быть уникальным в локальной сети. Прошивается производителем, но может быть изменён программно (MAC spoofing).|
|**Тип/Длина<br><br>(EtherType / Length)**|16 (2 байта)|Логика инкапсуляции: Указывает протокол верхнего уровня или длину данных.|**Если значение ≥ 1536 (0x0600)** - это **EtherType** (тип протокола):<br>• **0x0800** = IPv4<br>• **0x0806** = ARP<br>• **0x86DD** = IPv6<br>• **0x8100** = VLAN-тег (802.1Q)<br>• **0x88CC** = LLDP<br><br>**Если значение ≤ 1500** - это **длина** данных в байтах (старый формат 802.3).|
